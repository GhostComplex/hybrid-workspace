Build 的痛：组建团队难

一个 agent 不够用，新 agent 又上手慢

· 一个 Agent 跨多项目，context 不够用，性能下降
· 新 Agent 即使有 onboarding skill，实际项目中仍需反复沟通
· PM→QA 交接丢失决策上下文，组织记忆随 session 断裂
· 谁在做什么、卡在哪里，管理者无处可看

Train 的痛：教了记不住

Teach Once, Store Forever, Recall Never

· Skill 里写了「GitHub issue 图片用 markdown 格式」，但每次都忘
· 规则不是没有，是在 20+ 条中被上下文稀释——Agent 不会「触景生情」
· 同一条规则纠正多次，跨 session 后又忘
· 一个 Agent 学到的教训，同角色其他 Agent 不知道

Process 的痛：流程形同虚设

有规定没执行，人类失控

· QA 没写 test case 就验收，没截图就报 Pass
· Bug 和需求变更混着处理，该写 PRD 的直接开干
· 关键节点不等人类审批，Agent 自作主张往下走
· 人类说「停」，Agent 跑长任务时完全听不到

Architecture

Four Layers of Hybrid Workforce

Harness Layer

管理制度

一键创建 PM/Dev/QA，自带角色技能和行为红线。不是空白 Agent，是带着规矩上岗的员工。

Memory Layer

组织记忆

新 Agent 继承组织记忆，30 分钟上手而不是反复沟通几天。决策背景、团队习惯、踩过的坑自动传承。

Learning Layer

场景触发式召回

纠正一次永不再犯，不是存了就忘的假记忆。规则在关键时刻自动浮现，不靠 Agent 自己想起来。

Orchestration Layer

流程强制执行

人类定义流程，Agent 强制执行。跳步系统拦截，关键决策必须人类审批。

PHASE 1

Build Your AI Team

  • 团队模板——一键拉起 PM + Dev + QA
  • 角色定义——每个 Agent 有身份、准则、技能
  • 共享知识库——团队共享上下文，不重复交代

30 分钟从零到开始干活

PHASE 2

Teach Once, Recall Instantly, Remember Forever

Today: Teach Once, Store Forever, Recall Never.
Tomorrow: Teach Once, Recall Instantly, Remember Forever.

  • 场景触发式召回——Agent 执行工具前，系统自动注入精准规则（pre-tool-call hook）
  • 你说一句话 → 每个 Agent 自己识别相关规则 → 写入行为准则
  • 四层记忆保障：场景触发 / 行为前检查 / 失败记忆召回 / 渐进式强化
  • 不改模型，改 Harness——轻量、即时、可解释
PHASE 3

Process — 人类定义流程，Agent 必须遵守

  • 人类设计工作流程——QA 验收前必须写 test case、验收后必须出报告带截图
  • 区分 bug 和需求变更——不同问题走不同流程，需求变更需要 PRD 评审
  • 流程强制执行——Agent 试图跳步，系统自动拦截
  • Kill Switch——万一拦截没拦住，人类随时强制打断
  • Live Monitor——随时知道 Agent 在做什么、流程走到哪一步
